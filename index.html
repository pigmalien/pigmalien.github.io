"Thats how it always begins. Very small."

A favorite saying of mine is "You have to know what you can't do in order to know what you can."... or was that "You have to know what you can do in order to know what you can't? Ok maybe not favorite, but it still holds up.

In this blog I will attenpt to disseminate the simple mechanics of games. Take the simplest part of a new mechanic, and set it up to see how well it works, and if it would work well with other things like blocks fitting together.

I'm also going to talk about existing examples besides the experiments, as well as demo's that would would have a few of those parts strewn together. Then of course I'll be doing devblogs on current games, and other projects.

Side Note:

I'll be using Construct 3 for all the examples as it's as close as you can get to psudocode, and realy quick for prototyping.
